Chapter 01

The Vocabulary

Every term you've seen and never quite understood. Explained simply, with real-world analogies.

LLM
Large Language Model
A type of AI trained on massive amounts of text. It predicts what word/token comes next — and that simple trick, done at scale, creates something that sounds intelligent.
Analogy: Imagine someone who has read every book ever written. Ask them anything, they'll give you a pretty good answer — because they've seen similar patterns before.
API
Application Programming Interface
A way for your code to talk to someone else's software. You send a request, they send back a response. It's how you use AI models without building them yourself.
Analogy: A restaurant menu. You don't cook the food — you order from the menu (API), and the kitchen (server) prepares and returns it.
Parameters
aka "weights" or "B"
The numbers inside a neural network that define its behavior. When people say "7B" or "70B", they mean 7 billion or 70 billion parameters. More = usually smarter, but needs more RAM.
Analogy: Think of parameters like the dials on a mixing board. The more dials you have, the more fine-tuned and complex the sound (intelligence) can get.
Token
The unit of text AI thinks in
AI doesn't read words — it reads tokens. A token is roughly 3-4 characters or 0.75 of a word. "Hello world" = 2 tokens. APIs charge you per token. Context windows are measured in tokens.
Analogy: Like how music is broken into beats, not full songs. AI processes text beat by beat.
Context Window
aka "context length"
How much text the AI can "see" at once. If your conversation exceeds this limit, the AI forgets earlier parts. Measured in tokens. Claude has ~200K. GPT-4o has 128K.
Analogy: Like short-term memory. Once the window is full, older memories drop off the edge.
Inference
Running the model
When you actually use an AI model to generate output. Training is teaching the model. Inference is using it. Running Ollama locally = running inference on your machine.
Analogy: Training is going to school. Inference is actually doing your job using what you learned.
Prompt
Your input to the AI
The text you send to a model. A system prompt sets the AI's behavior/persona. A user prompt is your actual question. Good prompting = better outputs.
Analogy: A prompt is like a job brief. The clearer and more detailed it is, the better the result you'll get.
Temperature
Creativity dial (0.0 – 2.0)
Controls how random the AI's outputs are. 0 = deterministic, always picks the most likely next token. 2 = highly creative/random. For coding use 0–0.3. For creative writing use 0.7–1.2.
Analogy: Like the spice level at a restaurant. 0 = plain. 2 = ghost pepper. Pick what fits your needs.
Embedding
Text → Numbers
Converting text into a list of numbers (a vector) that represents meaning. Words with similar meaning have similar vectors. Used in search, RAG, and memory systems.
Analogy: Like GPS coordinates for words. "King" and "Queen" end up close together in space, because they're conceptually close.
RAG
Retrieval-Augmented Generation
Giving the AI access to external documents at runtime. Instead of just using its training data, it searches your docs and uses that context. This is how you build AI over your own data.
Analogy: Open-book exam vs closed-book. RAG lets the AI open the book (your data) while answering.
Fine-tuning
Specializing a model
Taking a pre-trained model and training it further on your specific data. Makes it better at a narrow task. Expensive and complex — most people use RAG instead.
Analogy: Hiring a generalist and sending them to a specialist bootcamp for your industry.
Quantization
Making models smaller
Compressing a model by reducing the precision of its numbers. A 7B model at Q4 uses ~4GB RAM. Same model at Q8 uses ~8GB. Lets you run big models on consumer hardware. GGUF is a common format.
Analogy: Like compressing a photo. Smaller file size, slightly lower quality, but still very usable.

Chapter 03

What Is an Agent?

An agent is an AI that doesn't just answer — it takes actions. It can use tools, make decisions, and loop until a task is done.

🤔
Simple Difference
Regular LLM: You ask → It answers → Done.

Agent: You give a goal → It thinks → It uses tools → It checks results → It tries again if needed → Done (eventually).
🤖 Inside an AI Agent
Inputs
💬User Goal / Task
🧠Memory (past context)
📄Documents / Data
🔄Previous results
→
🧠
THE AGENT
LLM at the core — it thinks, plans, decides
1. Read the goal
2. Decide what tool to use
3. Use the tool
4. Read the result
5. Repeat until done
→
Tools It Can Use
🔍Web search
💻Run code
📁Read/write files
🌐Call APIs
🗃️Query databases
🔁 The Agent Loop — How It Actually Thinks
1
Receive Goal
You give the agent a task: "Research the top 5 AI models of 2025 and write a summary report." This goes into its context window.
2
Think + Plan
The LLM reasons about the task. "I should search the web first. Then organize the results. Then write the report." This planning step is called chain-of-thought reasoning.
3
Use a Tool
Agent calls a tool — e.g., search the web. The tool runs and returns results back to the agent's context.
# Agent decides to use this tool:
tool: "web_search"
query: "top AI models 2025"
# Tool runs → results returned to agent
4
Observe + Decide
Agent reads the tool's output. "Okay I got results. Do I have enough? No, let me search for more." It keeps looping until it decides the task is done.
5
Output Final Answer
Once satisfied, the agent produces its final response. You get a full research report, not just a one-liner.

🖥️
Why Do People Run Multiple Agents in a Terminal?
Each window is running a different agent process. They each have their own LLM instance, their own tools, their own instructions. They communicate by passing messages (usually JSON). The orchestrator runs in one window and calls sub-agents in other windows. It looks complex but it's just multiple scripts talking to each other.

3

✓ All packages installed
4
Write Your First Agent
Runs entirely locally. No API key needed. No cost.
# agent.py — Your first local AI agent
from openai import OpenAI

# Point to LOCAL Ollama instead of OpenAI
client = OpenAI(
  base_url="http://localhost:11434/v1",
  api_key="ollama"
)

response = client.chat.completions.create(
  model="llama3.1:8b",
  messages=[
    {"role": "system", "content": "You are a helpful agent."},
    {"role": "user", "content": "Summarize AI trends in 2025"}
  ]
)

print(response.choices[0].message.content)
5
Run It
That's it. Local AI. Zero cost.
Terminal
$ python agent.py

In 2025, the major AI trends include...
multimodal models, local inference, agent frameworks...

✓ Ran locally. $0.00 spent. 🎉

Chapter 06

Your Learning Roadmap

Four phases. Go in order. By Phase 4 you'll be building things most people have no idea how to do.

Phase 01
Foundation
  • Install Ollama, pull llama3.1:8b
  • Chat with it in terminal
  • Learn what tokens, context, temperature mean
  • Write a Python script that calls the local API
  • Try Open WebUI for a ChatGPT-like interface
Phase 02
First Agent
  • Build a simple agent with 1–2 tools
  • Give it file read/write capability
  • Learn about system prompts and roles
  • Understand the agent loop (think → act → observe)
  • Try LangChain or OpenClaw framework
Phase 03
Multi-Agent
  • Build 2+ agents that talk to each other
  • Learn CrewAI or AutoGen
  • Add RAG — give agents access to your documents
  • Build an orchestrator + sub-agent setup
  • Add memory (short-term + long-term)
Phase 04
Production
  • Build a full workflow for a real use case
  • Add vector database (ChromaDB, Weaviate)
  • Fine-tune a small model on your own data
  • Build a UI around your agent system
  • Put it on GitHub — this is your portfolio
🎯
The Big Picture in One Sentence
An LLM is the brain. Ollama runs it locally. Tools give it hands. An Agent loop gives it persistence. A framework (OpenClaw, CrewAI) manages everything. Your Python code is the glue. And your local machine is powerful enough to run all of it for free.
